Three Libraries

医生库、药品库、医院库

这三类基础库构成了很多页面背后的事实层。医生相关页面用医生库，挂号或就医引导用医院库，治疗信息与禁忌提醒则要依赖药品库。

Doctor DB医生库

按姓名、医院、科室、职称、研究方向、代表论文和合作标签组织医生信息，适合医生主页和合作推荐页。

Common Outputs

这层数据通常会输出什么

基础库的价值不在“字段很多”，而在它可以稳定支持上层页面需要的卡片、筛选、推荐和路由动作。

Search Result检索卡片

适合搜索结果页、医生主页列表、药品说明结果和医院就医导航卡片。

Entity Profile实体详情页

支持医生详情、药品详情、医院详情和关联关系展示，例如擅长方向或挂号渠道。

Routing Layer场景路由层

给上层页面决定下一步跳到哪里，比如进入挂号流程、进入专科内容页或进入合作页面。

INTEGRATION

推荐的组合方式

基础库通常不会单独售卖，而是作为其他页面的“事实底座”。下面这个组合方式最容易讲清楚价值。

基础库Mock APISkill 层场景页
医生库 / 药品库 / 医院库
        -> 为搜索、卡片和实体详情提供基础事实
        -> 交给 Mock API 或 Skill 层补流程能力
        -> 最终被组装进挂号、知识服务、医院工作台或 Agent 页面
