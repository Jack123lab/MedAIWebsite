这里不只列官网，而是把 GitHub skills 的真实能力、适用场景和组合方法一起摆出来，便于快速从“能用什么”走到“怎么落地”。

精选技能OpenClaw 医学技能例子
medical-research-toolkit统一 MCP 入口连接 14+ 生物医学数据库，适合药物发现、证据检索与疾病研究。 biomcp-server把 PubMed、PMC、PubTator3、ClinicalTrials.gov 等能力标准化成 MCP 服务。 tooluniverse-clinical-guidelines面向 NICE、WHO、ADA、AHA、NCCN 等 12+ 指南来源的搜索与引用。 digital-twin-clinical-agent构建患者数字孪生，用于治疗响应预测、临床试验模拟和个体化医学。 trialgpt-matching把患者画像映射为临床试验 shortlist，并输出缺失信息与排序解释。
场景地图工具与 MCP 产品方向
循证医疗 API多数据库、多指南、多证据等级的检索与引用层。 虚拟医院面向病人数字孪生、病例回放、医学生训练与模拟验证。 Agentic hospital把分诊、病历总结、检验追踪、随访和决策支持串成代理链。 AI-native HIS将 FHIR/EHR、病历 Copilot 和结构化数据流对接到医院信息系统。 AI-native Up2date做类 UpToDate 的持续更新型指南与文献助手。 中医知识图谱连接证候、方剂、药材、症状与现代知识图谱推理。 医生 / 药品 / 医院库把基础库检索和 mock 挂号 API 接到医生主页、医院主页和问答助手。
落地方式从 skill 到产品的三层组合
  1. 第一层：工具与 API接入 PubMed、ClinicalTrials.gov、OpenFDA、FHIR、知识图谱和医学指南等基础能力。
  2. 第二层：代理工作流用 summarization、care coordination、trial matching、clinical decision support 组织任务链。
  3. 第三层：产品界面包装成医生 Copilot、虚拟医院、AI-native HIS、AI-native Up2date 或中医 Graph-RAG 产品。

Scenario Stack

工具与 MCP 场景栈

每个方向都给出可直接引用的 skill 例子，帮助把 GitHub 仓库里的能力转成网站上的可理解产品模块。

循证医疗循证医疗 API

围绕多数据库检索、指南搜索和临床证据引用，适合做 evidence copilot、院内文献助手和标准化问答接口。

PubMedClinicalTrialsOpenFDANICE / WHO / NCCN
medical-research-toolkit统一 MCP 端点聚合 ChEMBL、PubMed、ClinicalTrials.gov、OpenTargets、OpenFDA、OMIM 等来源。 tooluniverse-clinical-guidelines跨 NICE、WHO、ADA、AHA、NCCN、GIN、TRIP 等 12+ 权威指南源做检索与回引。 biomcp-server标准化暴露 PubMed/PMC、PubTator3、ClinicalTrials.gov 与变异查询能力。

例子：给“重症肌无力免疫治疗”输出带 PubMed、ClinicalTrials.gov、OpenFDA 引用的证据摘要与正在招募的试验列表。

Simulation虚拟医院

面向病例回放、患者数字孪生、训练环境和多角色协作，适合教学、方案模拟和新流程验证。

Patient twinEHR chatVirtual lab
digital-twin-clinical-agent基于临床记录、多组学与影像构建患者数字孪生，用于反事实治疗模拟。 chatehr-clinician-assistant让临床人员通过自然语言检索病历、汇总病程并生成出院摘要。 virtual-lab-agent组织 PI、免疫学家、计算生物学家和 ML 专家等多 agent 团队做实验设计。

例子：做一个“虚拟病区”，让医生或学生先在数字孪生患者上试跑治疗方案、病程总结和多学科讨论。

WorkflowAgentic hospital

把分诊、病历整理、检验结果追踪、照护协调和决策支持做成一条自动接力的医院代理流水线。

SummarizationCoordinationCDSLabs
clinical-note-summarization把原始病历结构化成 SOAP，总结矛盾点、缺失项和 ICD 关联判断。 care-coordination适合接管转诊、会诊、随访和跨科室任务编排。 lab-results聚焦检验结果解释与工作流联动，适合异常结果追踪和触发提醒。 clinical-decision-support面向证据等级、治疗建议和临床决策报告的生成。

例子：门诊分诊后自动生成 SOAP，拉取异常化验，整理随访任务，再把决策支持结果推给医生复核。

SystemAI-native HIS

把 FHIR、EHR、病历 Copilot 和结构化数据服务接到 HIS 中，让 AI 不只是外部助手，而是系统原生能力。

FHIREHRCopilot
ehr-fhir-integration通过 HL7 FHIR 对患者、诊断、用药、检验等资源做检索、读取和导出。 chatehr-clinician-assistant把自然语言查询、图表摘要和病历问答嵌到临床工作站。 clinical-note-summarization可作为 HIS 中病历结构化、质控和编码前处理的一层。

例子：在 HIS 里直接问“最近一次肌酐是多少”“总结最近 3 次肿瘤科门诊”，系统返回结构化答案并保留审计链。

KnowledgeAI-native Up2date

做一个持续更新、自动带来源、支持指南和最新研究并读的 AI-native 临床知识系统，而不只是静态文档库。

GuidelinesResearch lookupDaily brief
tooluniverse-clinical-guidelines提供权威指南层，适合查询标准治疗、筛查策略和药物基因组推荐。 medical-specialty-briefs按专科生成每日或按需更新的研究简报，适合持续追踪高影响力论文。 research-lookup实时查询最新论文、技术文档和当前研究趋势，适合补足“最新研究”层。

例子：给心内科做一个“AI-native Up2date”，每天自动整理过去 7 天的新指南、RCT、JAMA/NEJM 文章并生成一句话临床 takeaway。

TCM中医知识图谱

把中医体质、证候、症状、方剂、药材与现代知识图谱推理结合，做成适合教育、研究和临床知识检索的 Graph-RAG 模块。

体质辨识Graph-RAG多跳推理
tcm-constitution-analyzer适合做中医体质辨识、养生建议和体质特征分析的起点模块。 kragen-knowledge-graph提供 Graph-of-Thoughts 风格的知识图谱增强 RAG，可扩展到证候与方剂关系推理。

例子：构建“证候 - 症状 - 方剂 - 药材 - 现代机制”五层图谱，用 Graph-RAG 回答复方配伍和疾病机制的多跳问题。

Examples

展示例子

下面这些例子可以直接作为页面里的“可做什么”说明，也方便后续继续扩成 demo、MCP 网关或产品介绍。

Example 01Evidence Copilot

输入疾病名后，同时检索 PubMed、ClinicalTrials.gov、OpenFDA 和主流指南，输出证据摘要、试验列表和风险信号。

组合：medical-research-toolkit + tooluniverse-clinical-guidelines + biomcp-server

Example 02Virtual Hospital Sandbox

用数字孪生患者、病历聊天和多 agent 实验设计，做住院流程、MDT 或医学生训练的模拟环境。

组合：digital-twin-clinical-agent + chatehr-clinician-assistant + virtual-lab-agent

Example 03Agentic Ward Coordinator

接收原始病历和检验结果后，自动输出 SOAP、异常提醒、照护协调任务和决策支持建议。

组合：clinical-note-summarization + lab-results + care-coordination + clinical-decision-support

Example 04AI-native HIS Layer

将 FHIR 数据、病历问答和结构化摘要做成 HIS 的原生智能层，支持临床查询、编码和审计留痕。

组合：ehr-fhir-integration + chatehr-clinician-assistant + clinical-note-summarization

Example 05AI-native Up2date

每天按专科自动追踪最新研究与指南，生成带来源链接的一句话总结和临床 relevance 标签。

组合：tooluniverse-clinical-guidelines + medical-specialty-briefs + research-lookup

Example 06TCM Graph-RAG

将中医体质、症状、证候和方剂连成图谱，再结合现代医学实体做跨体系多跳问答和知识发现。

组合：tcm-constitution-analyzer + kragen-knowledge-graph
