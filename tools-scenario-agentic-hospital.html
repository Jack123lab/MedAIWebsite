Agentic Hospital

这不是“一个超强医生助手”，而是一组在医院流程里分工协作的代理系统，负责把分诊、病历整理、检验追踪、照护协调和决策支持串成一条自动接力的服务链。

Agent Roles

五个最关键的代理角色

完整页面的重点，是把“agentic”具体化为清晰的角色职责，而不是一句空泛的多智能体协作。

01triage agent

接收主诉和基本信息，判断分流方向、紧急程度和需要优先补齐的数据。

02note agent

把原始病历、口述记录和外部资料整理成 SOAP 或结构化摘要。

03lab agent

跟踪化验和检验结果，识别异常、提醒复查并推动后续任务触发。

04care coordination agent

负责随访、会诊、转诊、患者提醒和多科室之间的任务交接。

05decision support agent

调用指南和证据层，为医生提供待复核的治疗建议、风险提醒和下一步检查建议。

Governance

什么时候适合做，什么时候不能越界

多代理页面最容易被误解成“自动化替代医生”，所以边界写法必须直接而明确。

Write-back自动写回必须受控

任何自动写回 HIS 或任务系统的动作，都必须有明确责任人和回滚策略。

Human Gate关键决策节点保留人工门

涉及诊断、治疗、出院和会诊升级的节点，代理只给建议，不直接执行。

Trace每个代理都要留痕

需要记录哪个代理做了什么、用到了哪些输入、触发了什么输出，便于审核。
